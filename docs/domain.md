# Доменные сущности

## Владелец

Владелец календаря — один заранее заданный профиль, который используется в админской части.

Свойства:
- `id`
- `name`
- `email` (опционально)

Поведение:
- создание и редактирование типов событий
- просмотр списка всех бронирований

## Тип события

`EventType` описывает доступный вид встречи.

Свойства:
- `id`
- `name`
- `description`
- `durationMinutes`
- `ownerId`

Особенности:
- каждый тип события задает название, описание и длительность
- используется гостем для выбора вида брони

## Слот

`Slot` описывает конкретное время для записи.

Свойства:
- `eventTypeId`
- `startAt`
- `endAt`
- `status` (`free` / `booked`)

Правила:
- слоты формируются на ближайшие 14 дней от текущей даты
- гость может выбрать только свободный слот
- слот не должен пересекаться с другими бронированиями

## Бронирование

`Booking` фиксирует реальную запись гостя.

Свойства:
- `id`
- `eventTypeId`
- `slotStart`
- `slotEnd`
- `guestName` / `guestContact` (если необходимо)
- `createdAt`

Правила:
- бронирование связано с типом события и занимает слот
- нельзя создавать две записи на одно и то же время
- бронь подтверждается при выборе свободного слота

## Публичный сценарий гостя

1. Открывает страницу доступных типов событий.
2. Видит для каждого типа название, описание и длительность.
3. Выбирает нужный тип события.
4. Открывает календарь свободных слотов на ближайшие 14 дней.
5. Выбирает свободный свободный слот.
6. При необходимости заполняет данные гостя.
7. Создает бронирование.
8. Система проверяет, что слот свободен, не пересекается с другими записями и находится в окне 14 дней.
